vs code

기본 설정

  • 기본적인 vs code의 공백은 '2'로 설정한다.
  • Prettier 확장 프로그램을 사용하여, 코드의 통일성을 맞춘다.

유용한 vs code 확장 프로그램*필수

  • Prettier (코드 자동 나열)
  • Git Graph (브랜치를 직관적인 그래프로 보여주는 확장 프로그램)
  • Bracket Pair Color DLW (열림태그와 닫힘태그의 구분을 컬러로 쉽게 보여주는 확장 프로그램)
  • BEM Helper (BEM 방법론 유효성 여부 체크를 도와주는 확장 프로그램)
  • Auto Rename Tag (태그의 수정이 있을 경우 자동으로 열림태그와 닫힘태그 둘다 수정을 도와줌)

class naming

네이밍 규칙

  • 모든 class 네이밍 규칙은 BEM(Block Element Modifier) 방법론에 의해 작성되어야 한다. link
  • 일반적인 블록,엘리먼트가 아닌 경우 접두사를 이용해 기능의 정의를 확실히 구분한다. link
  • 전체 공통(common) class를 수정 및 추가 할 경우, 반드시 프로젝트 팀원에게 알려야 한다.(slick 또는 kakao)
  • 개인 페이지 작업 시 최상단의 class명은 main태그의 class를 따라야한다.(중복 방지) //html <main class="exname"> <div class="content"> <p class="content__title">hello</p> </div> </main> //sass .exname{ .content{ &__title{} } }
  • 한 페이지에 여러명의 작업자가 붙을 경우, 해당 페이지에서 공통으로 사용할 class명은 각 작업자와의 회의를 통해 정한다.
  • 기본적으로 common에 구성되어있는 img-box / sr-only / mo-only / pc-only 등 활용한다.
  • 자신만 알아볼 수 있는 class명이 아닌 '명확한 목적성'을 가져야 한다. (ex: main__title,main__list,main__title--large / 잘못된 ex: main__jhstyle, addstyle-01)

git

git 브랜치

  • git 브랜치는 main / develop 으로 구성 되어있다. main : 최종 QA까지 마친 작업물 브랜치 , develop : 퍼블리싱 작업의 가장 최신버전
  • 개인 페이지 작업시에는 develop 브랜치에서 새 브랜치를 생성 후 작업한다.
  • 커밋 메세지는 '페이지명 : 내용' 으로 명확하게 기재한다.
  • 한 페이지에 여러 작업자가 작업 할 경우 머지 전에 수정 내역 공유 후 머지 한다(충돌 방지)
  • develop에 머지 후에 프로젝트 작업자 전원에게 머지 완료 안내를 해야한다.

접근성

접근성에 관하여

  • img태그의 alt 속성은 이미지내 텍스트가 존재할 때 필히 기재한다.
  • 동작이 들어가는 button은 button태그를 사용하여 작성한다.
  • 목적에 맞는 마크업 태그를 사용한다.
  • 이미지,아이콘에 설명 또는 타이틀이 필요한 경우 <span class="sr-only">아이콘</span> 해당 방식으로 접근성을 준수한다.
  • on/off 기능이 있는 radio 또는 checkbox가 있을 경우 선택이 완료되었을 때 <span class="sr-only">선택됨</span> 해당 방식으로 선택이 되었음을 표기한다.
  • 독립 태그는 뒤에 태그닫힘을 표기한다. <img src="" alt=""/> <br/> (해당 방식을 준수하는 이유는 개발 환경으로 넘어갔을 때 문법을 맞추기 위함)
